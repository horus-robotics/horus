# Installing HORUS

Get HORUS up and running in 5 minutes. We'll install Rust, build HORUS, and verify everything works.

## Platform Support

HORUS works on systems with POSIX shared memory support:

| Platform | Status | Notes |
|----------|--------|-------|
| **Ubuntu 20.04+** | ✅ Fully Supported | Recommended for production |
| **Ubuntu 22.04+** | ✅ Fully Supported | Best performance |
| **Debian 11+** | ✅ Supported | Tested and working |
| **Fedora 36+** | ✅ Supported | Use dnf for packages |
| **Arch Linux** | ✅ Supported | Community maintained |
| **macOS 11+** | ✅ Supported | Limited shared memory size |
| **Windows** | ⚠️ Via WSL Only | Use WSL 2 for best results |
| **Raspberry Pi** | ✅ Supported | ARM64 tested on Ubuntu |

## Prerequisites

**Required:**
- **Operating System**: Linux (Ubuntu 20.04+) or macOS 11+
- **Rust 1.70+**: We'll install this in Step 1
- **Build Tools**: C compiler, pkg-config, and system libraries
  ```bash
  # Ubuntu/Debian - COMPLETE dependencies (copy-paste this!)
  sudo apt update && sudo apt install \
    build-essential \
    pkg-config \
    libudev-dev \
    libssl-dev \
    libasound2-dev

  # Fedora/RHEL
  sudo dnf groupinstall "Development Tools"
  sudo dnf install pkg-config systemd-devel openssl-devel alsa-lib-devel

  # Arch Linux
  sudo pacman -S base-devel pkg-config systemd openssl alsa-lib

  # macOS
  xcode-select --install
  brew install pkg-config openssl
  ```
- **10-15 minutes**: For first-time installation
- **Internet connection**: To download dependencies

**Don't worry if you don't have:**
- Rust (we'll install it)
- Git (we'll install it)
- Experience with systems programming (not needed!)

## Quick Install (Recommended)

Copy and paste these commands into your terminal:

```bash
# 1. Install Rust (takes ~2 minutes)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env

# 2. Clone HORUS
git clone https://github.com/neos-builder/horus.git
cd horus

# 3. Run automated installer (takes ~5 minutes)
./install.sh

# 4. Verify it works
horus --version
```

If you see `horus 0.3.0` (or similar), you're all set! Skip to [Next Steps](#next-steps).

**What the installer does:**
- Builds the `horus` CLI tool
- Installs six core libraries to `~/.horus/cache/`:
  - `horus_core` - Core runtime and scheduler
  - `horus` - Main framework library
  - `horus_macros` - Procedural macros for simplified syntax
  - `horus_library` - Standard message types and nodes
  - `horus_c` - C bindings with headers
  - `horus_py` - Python bindings
- Tracks the installed version for automatic updates
- Verifies installation with built-in tests

## Step-by-Step Installation

### Step 1: Install Rust

HORUS is built with Rust. Don't worry if you've never used it - you don't need to know Rust to use HORUS (Python and C work too!).

**On Linux or macOS:**
```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

Follow the prompts (just press Enter for defaults).

**Restart your terminal or run:**
```bash
source $HOME/.cargo/env
```

**Verify Rust is installed:**
```bash
rustc --version
cargo --version
```

You should see version numbers like `rustc 1.75.0`.

**On Windows:**
Use Windows Subsystem for Linux (WSL):
```powershell
# In PowerShell (as Administrator)
wsl --install

# Then follow Linux instructions inside WSL
```

### Step 2: Install Git (If You Don't Have It)

**Check if you have Git:**
```bash
git --version
```

**If not, install it:**
```bash
# Ubuntu/Debian
sudo apt install git

# macOS
xcode-select --install

# Fedora
sudo dnf install git
```

### Step 3: Clone HORUS

Download the HORUS source code:

```bash
git clone https://github.com/neos-builder/horus.git
cd horus
```

This creates a `horus` directory with all the code.

### Step 4: Run the Installer

Use the automated installer to build and install everything:

```bash
./install.sh
```

This script:
1. Builds all HORUS packages (takes ~5 minutes)
2. Installs the `horus` CLI to `~/.cargo/bin/`
3. Installs core libraries to `~/.horus/cache/`
4. Saves version information for updates
5. Runs verification tests

You'll see colored output showing progress - green checkmarks mean success!

### Step 5: Verify Installation

Test that the `horus` command works:

```bash
horus --version
```

You should see:
```
horus 0.3.0
```

Try the help command:
```bash
horus --help
```

You should see a list of available commands like `new`, `run`, `dashboard`, etc.

## Optional: Python Support

Want to use HORUS with Python? Here's how:

```bash
# 1. Install maturin (Python build tool)
pip install maturin

# 2. Navigate to Python bindings
cd horus_py

# 3. Build and install (takes ~3 minutes)
maturin develop --release

# 4. Test it works
python -c "import horus; print('Python bindings work!')"
```

See [Multi-Language](/multi-language) for Python examples.

## Optional: C Support

Want to integrate C code? Build the C bindings:

```bash
cd horus_c
cargo build --release
```

The shared library will be in `target/release/`:
- `libhorus_c.so` (Linux)
- `libhorus_c.dylib` (macOS)

See [Multi-Language](/multi-language) for C examples.

## Platform-Specific Notes

### Linux (Ubuntu/Debian)

Most things "just work" on Linux. If you hit build errors:

```bash
sudo apt update && sudo apt install \
  build-essential \
  pkg-config \
  libudev-dev \
  libssl-dev \
  libasound2-dev
```

### Linux (Fedora/RHEL)

```bash
sudo dnf groupinstall "Development Tools"
```

### macOS

Install Xcode Command Line Tools if you haven't:

```bash
xcode-select --install
```

**Note:** macOS has smaller shared memory limits than Linux. For most applications this is fine, but if you're moving lots of large messages, Linux is better.

### Windows

Use WSL (Windows Subsystem for Linux):

1. Open PowerShell as Administrator
2. Run: `wsl --install`
3. Restart your computer
4. Follow the Linux installation steps inside WSL

## Understanding Shared Memory

HORUS uses `/dev/shm` for ultra-fast communication between components.

**Check available space:**
```bash
df -h /dev/shm
```

You should have at least 256MB. Most systems have 1-2GB.

**If you need more space (advanced):**
```bash
# Temporarily increase to 2GB
sudo mount -o remount,size=2G /dev/shm

# Make permanent: edit /etc/fstab (requires sudo)
# Add line: tmpfs /dev/shm tmpfs defaults,size=2G 0 0
```

## Updating HORUS

To update to the latest version:

```bash
# Navigate to HORUS directory
cd horus

# Pull latest changes
git pull

# Run installer again (auto-detects version changes)
./install.sh
```

The installer automatically:
- Detects version changes
- Removes old library versions
- Rebuilds and reinstalls everything
- Updates the CLI tool

You'll see a message like: `WARNING: Version changed: 0.3.0 → 0.3.1`

## Uninstalling

To completely remove HORUS:

```bash
# Navigate to HORUS directory
cd horus

# Run the uninstaller
./uninstall.sh
```

This removes:
- The `horus` CLI command
- All cached libraries (`~/.horus/cache/`)
- Version tracking files
- Shared memory files (`/dev/shm/horus/`)

**Manual uninstall (if needed):**
```bash
# Remove CLI tool
cargo uninstall horus

# Remove global cache
rm -rf ~/.horus/

# Remove source code
rm -rf ~/horus/  # or wherever you cloned it

# Clean up shared memory
rm -rf /dev/shm/horus/
```

## Troubleshooting

### Quick Fixes

**"command not found: horus"**
```bash
# Add to your shell config (~/.bashrc or ~/.zshrc)
export PATH="$HOME/.cargo/bin:$PATH"
source ~/.bashrc  # or ~/.zshrc
```

**Build errors ("linker not found", "pkg-config", etc.)**
```bash
# Ubuntu/Debian - Install ALL required dependencies
sudo apt update && sudo apt install \
  build-essential \
  pkg-config \
  libudev-dev \
  libssl-dev \
  libasound2-dev

# Fedora/RHEL
sudo dnf groupinstall "Development Tools"
sudo dnf install pkg-config systemd-devel openssl-devel alsa-lib-devel

# macOS
xcode-select --install
brew install pkg-config openssl
```

**Installation broken or corrupted**
```bash
cd horus
./recovery_install.sh  # Diagnose and fix issues
```
