## Running Examples

All examples are self-contained and can be run directly:

### Single File (Rust)
```bash
# Copy example code to a file
nano my_example.rs

# Run it
horus run my_example.rs
```

### Single File (Python)
```bash
# Create Python file
nano my_example.py

# Run it
horus run my_example.py
```

### Multi-Process (Multiple Files)
```bash
# Create directory with multiple node files
mkdir nodes/
# ... create nodes/sensor.py, nodes/controller.py, etc.

# Run all concurrently
horus run "nodes/*.py"

# Or mix Rust and Python
horus run "nodes/*"
```

## Tips for Success

### Writing Your Own Examples

1. Start simple - Get a basic publisher-subscriber working first
2. Add one feature at a time - Don't try to build everything at once
3. Test frequently - Run `horus run` after each change
4. Use the dashboard - `horus dashboard` helps visualize message flow
5. Check the logs - Enable debug logging with `RUST_LOG=debug`

### Common Patterns

Pipeline: `Sensor  Filter  Controller  Actuator`
- Data flows one direction
- Each node has increasing priority (0, 1, 2, 3)

Aggregator: `Sensor1, Sensor2, Sensor3  Fusion  Output`
- Multiple inputs, single output
- Use `if let Some(msg) = hub.recv(ctx)` for bounded execution (one message per tick)

Safety Override: `Critical Safety (priority 0)  Control (priority 1)`
- Safety checks always run first
- Control nodes check estop before acting
