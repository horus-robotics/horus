# Installation

This guide will walk you through installing HORUS on your system and getting it ready for development.

## Prerequisites

Before installing HORUS, ensure you have the following installed:

**Required:**
- **Rust** (version 1.70 or later)
- **Cargo** (comes with Rust)
- **Git**

**Optional but Recommended:**
- **Python 3.8+** (for Python bindings)
- **C Compiler** (GCC or Clang, for C bindings)

## Quick Install

The fastest way to get started with HORUS:

```bash
# Clone the repository
git clone https://github.com/neos-builder/horus
cd horus

# Run the installation script (recommended)
./install.sh

# Verify installation
horus --version
```

**What the installer does:**
- Builds all packages in release mode
- Installs `horus` CLI to `~/.cargo/bin/`
- Sets up library cache in `~/.horus/cache/`
- Creates package metadata and handles versions automatically

## Detailed Installation Steps

### Step 1: Install Rust

If you don't have Rust installed:

```bash
# On Linux/macOS
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Follow the prompts and restart your terminal
source $HOME/.cargo/env

# Verify installation
rustc --version
cargo --version
```

### Step 2: Clone HORUS

```bash
git clone https://github.com/neos-builder/horus
cd horus
```

### Step 3: Run Installation Script

The installation script handles everything automatically:

```bash
# Run the installer
./install.sh
```

**The script will:**
1. Check that Rust is installed
2. Build all packages in release mode (`cargo build --release`)
3. Install the `horus` CLI to `~/.cargo/bin/`
4. Set up `~/.horus/cache/` with versioned libraries:
   - `horus_core@0.1.0` - Core runtime and scheduler
   - `horus@0.1.0` - Main framework library
   - `horus_macros@0.1.0` - Procedural macros for simplified syntax
   - `horus_library@0.1.0` - Standard message types and nodes
   - `horus_c@0.1.0` - C bindings with headers (automatically available)
   - `horus_py@0.1.0` - Python bindings (automatically available)
5. Create `metadata.json` for each package
6. Verify the installation
7. Display next steps

**Manual Installation (Alternative):**

If you prefer manual control:

```bash
# Build everything
cargo build --release

# Install CLI only
cargo install --path horus_manager

# Note: You'll need to manually set up ~/.horus/cache/
```

### Step 4: Verify Installation

Check that everything works:

```bash
# Check version
horus --version

# Check library cache
ls ~/.horus/cache/

# List available commands
horus --help
```

## Platform-Specific Notes

### Linux

HORUS uses shared memory (`/dev/shm`) which is available on all modern Linux distributions. No special configuration needed.

**Ubuntu/Debian:**
```bash
# Install build essentials if needed
sudo apt-get update
sudo apt-get install build-essential pkg-config
```

**Fedora/RHEL:**
```bash
# Install development tools
sudo dnf groupinstall "Development Tools"
```

### macOS

HORUS works on macOS with some limitations:

- Shared memory is available but may have size limitations
- Performance may vary compared to Linux

```bash
# Install Xcode Command Line Tools
xcode-select --install
```

### Windows (WSL Recommended)

For Windows, we recommend using **Windows Subsystem for Linux (WSL)**:

```bash
# Install WSL2 (PowerShell as Administrator)
wsl --install

# Then follow Linux installation instructions inside WSL
```

## Python Bindings (Optional)

To use HORUS with Python:

```bash
# Navigate to Python bindings
cd horus_py

# Build and install
maturin develop --release

# Verify Python bindings
python -c "import horus; print(horus.__version__)"
```

## C Bindings (Optional)

To use HORUS with C:

```bash
# Build C bindings
cd horus_c
cargo build --release

# The shared library will be in target/release/
# libhorus_c.so (Linux) or libhorus_c.dylib (macOS)
```

## Environment Setup

### Shell Configuration

Add HORUS binaries to your PATH (if not already added by cargo install):

```bash
# Add to ~/.bashrc or ~/.zshrc
export PATH="$HOME/.cargo/bin:$PATH"
```

### Shared Memory Configuration

HORUS uses `/dev/shm` for shared memory IPC. Check available space:

```bash
df -h /dev/shm
```

If you need more space (Linux):

```bash
# Increase to 2GB (requires sudo)
sudo mount -o remount,size=2G /dev/shm
```

## Updating HORUS

To update to the latest version:

```bash
# Pull latest changes
cd horus
git pull

# Re-run installer (handles version upgrades automatically)
./install.sh
```

**What happens during update:**
- Detects version changes
- Removes old library versions if needed
- Installs new versions to `~/.horus/cache/`
- Preserves your existing configuration

**Manual Update (Alternative):**
```bash
# Pull changes
git pull

# Rebuild
cargo build --release

# Force reinstall CLI
cargo install --path horus_manager --force
```

## Uninstalling

To remove HORUS completely:

```bash
# Use the uninstall script (recommended)
cd horus
./uninstall.sh
```

**What the uninstaller does:**
- Removes CLI binary from `~/.cargo/bin/horus`
- Removes global library cache from `~/.horus/cache/`
- Optionally removes entire `~/.horus/` directory (asks first)
- Cleans shared memory files in `/dev/shm/horus/`
- Preserves project-local `.horus/` directories

**Manual Uninstall (Alternative):**
```bash
# Remove CLI
cargo uninstall horus

# Remove cache
rm -rf ~/.horus/cache/

# Optionally remove all HORUS config
rm -rf ~/.horus/

# Remove source directory
rm -rf horus

# Clean shared memory
rm -rf /dev/shm/horus/
```

## Common Issues

### "command not found: horus"

**Solution:** Make sure `~/.cargo/bin` is in your PATH:

```bash
export PATH="$HOME/.cargo/bin:$PATH"
source ~/.bashrc  # or ~/.zshrc
```

### Build fails with "linker not found"

**Solution:** Install a C compiler:

```bash
# Ubuntu/Debian
sudo apt-get install build-essential

# macOS
xcode-select --install
```

### Installation broken or corrupted

**Solution:** Run recovery install:

```bash
./recovery_install.sh  # Diagnose and fix issues
```
